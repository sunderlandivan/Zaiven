Order tracking

Gmail order sync

Reads TCGplayer seller emails and USPS label mail, then merges by order number when possible. USPS Click‑N‑Ship uses its own order id (UUID), not your TCGplayer order # — we match those labels to rows by buyer name + label date + postage vs shipping, so a wrong order date in the CSV or a crowded inbox can still miss a row. Use Sync Gmail (full YTD) when you want a deeper pass (slower). Mail is limited to on or after Jan 1, 2026 unless GMAIL_SYNC_AFTER is set.

Import TCGplayer orders CSV

CSV import captures order number, buyer, date, status, amounts, and tracking when the export includes it. If the sheet has a Tax / Sales Tax column, it is stored as buyer tax for fee estimates. Many TCGplayer order-list CSVs leave carrier/tracking blank — use Gmail sync for tracking; re-importing the same CSV keeps existing tracking when the file still has no carrier column.

Record an order

For TCGplayer rows with blank fees, estimates follow TCGplayer’s published fee types (banker’s rounding). Default is Level 1–4 (10.75% on subtotal with a $75/unit cap when it applies, plus 2.5% + $0.30 on the charged total including buyer tax). If your portal totals look like Marketplace Pro (9.25% + 2.5% on subtotal plus the same transaction fee), set TCG_FEE_SELLER_PROFILE=pro on the server. If buyer tax is unknown, the server uses TCG_ESTIMATED_BUYER_TAX_RATE (7% for Level 1–4, 8.5% when profile is Pro); override the env var or enter buyer tax when you know it.
